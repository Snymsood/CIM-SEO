# Empty Charts Fix - RESOLVED

## Problem
The HTML report generated by `gsc_landing_pages_report.py` had empty chart sections when viewed in a browser.

## Root Causes Identified

### 1. Function Order Issue
The helper functions `_img_tag()`, `_chart_wrap()`, and `_chart_row_2()` were defined AFTER `write_html_summary()` but were being called inside the f-string template. This caused them to be undefined when the template was evaluated.

**Solution**: Moved these helper functions to be defined BEFORE `write_html_summary()`.

### 2. Path Return Issue  
The `_save()` function was returning `f"file://{path.absolute()}"` instead of just the `Path` object. This caused the HTML to contain `file:///` URLs which browsers cannot load.

**Solution**: Changed `_save()` to return just `path` instead of the file URL.

### 3. Workflow Branch Issue
The GitHub Actions workflow was running on the `main` branch instead of the feature branch `update-landing-pages-report-html-only` where the fixes were made.

**Solution**: Triggered the workflow with `--ref update-landing-pages-report-html-only` flag.

## Pattern Consistency
The fix ensures `gsc_landing_pages_report.py` follows the same pattern as `gsc_weekly_report.py`:
1. `_save()` returns a `Path` object
2. `_img_tag()` converts it to a string for the HTML `src` attribute  
3. `embed_images_as_base64()` reads the file and converts to base64 data URI
4. Final HTML is self-contained with all images embedded inline
