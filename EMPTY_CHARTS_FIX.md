# Empty Charts Fix - Landing Pages Report

## Problem
The HTML report generated by `gsc_landing_pages_report.py` was being uploaded to Monday.com successfully, but all charts appeared empty in the browser.

## Root Cause
The `_save()` function in `gsc_landing_pages_report.py` was returning a string with the `file://` protocol:

```python
return f"file://{path.absolute()}"
```

This caused two issues:
1. The HTML contained `<img>` tags with `src="file:///absolute/path/to/chart.png"` which browsers cannot load (security restriction)
2. The `embed_images_as_base64()` function's regex pattern `r'src="([^"]+\.(png|jpg|jpeg|svg|gif))"'` would match the path, but when it tried to read the file using `Path(src)`, it would fail because `src` included the `file://` protocol

## Solution
Changed the `_save()` function to return just the `Path` object, matching the pattern used in `gsc_weekly_report.py`:

```python
return path
```

This allows:
1. The `_img_tag()` function to insert the path as a simple string (e.g., `charts/landing_page_comparison.png`)
2. The `embed_images_as_base64()` function to properly read the file and convert it to a base64 data URI

## Pattern Consistency
This fix ensures `gsc_landing_pages_report.py` follows the same pattern as `gsc_weekly_report.py` for image handling:
1. `_save()` returns a `Path` object
2. `_img_tag()` converts it to a string for the HTML `src` attribute
3. `embed_images_as_base64()` reads the file and converts to base64 data URI
4. Final HTML is self-contained with all images embedded inline
